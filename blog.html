Question-1: What is DOM and what is the purpose of DOM?

DOM means Document Object Model. DOM in JavaScript is a way to access and change the content and structure of a web page. It allows developers to use JavaScript to interact with the elements of an HTML or XML document.

The DOM in JavaScript serves as a tool for developers to dynamically alter a web page's content, structure, and style in response to events or actions taken by users. In order to create dynamic and interactive web pages, JavaScript may access and edit the properties and attributes of specific HTML elements, as well as inserting or removing items from the page and changing their appearance.

For example, with the DOM, developers can change the text or images on a web page, add new elements to the page, or make elements respond to user actions.

The DOM is a fundamental part of creating dynamic and interactive web pages and enables developers to make web pages that are more engaging and useful for users.

Question-2: How will you select HTML elements using DOM? Name the DOM methods.

To select HTML elements using the DOM in JavaScript, we can use different methods depending on the specific element or elements we want to select. Name of some DOM methods with simple explanation about their functionality is given in the following-

Some DOM Methods for Selecting HTML Elements:

  • getElementById(): This method returns an element whose id matches a passed string.
  • getElementsByTagName(): This method returns the HTMLCollection of elements that match the passed tag name.
  • getElementsByClassName(): This method returns an HTMLCollection of elements that match the passed class name.
  • getElementsByName(): This method returns the elements that match the value of the name attribute with the passed string. The return value is a live NodeList Collection.
  • querySelector(): This method returns the first element that matches the passed selector.
  • querySelectorAll(): This method is an extension of the querySelector method. This method returns all the elements that match the passed selector.

Question-3: What is event bubble?

Event bubbling in JavaScript allows a child element's event to be transmitted up through its parent elements until it reaches the top-level HTML content.

Since the event handlers are called in order from the innermost to the outermost element, this enables many elements to react to the same event.

The event "bubbles up" across the DOM structure from the child element to its parent elements, that's the reason it is called "bubbling."

Question-4: What is a callback function and why will you use it?

A callback function is a function that is passed as an argument to another function and is called by that function when a certain event or condition occurs. The purpose of a callback function is to allow certain parts of the code to be executed without blocking the execution of other parts of the code.

Simply, a callback function is a function that is called by another function when a certain action is completed. It can be used to perform some operation after a certain task is completed, or it can be used to execute a function after a certain event occurs. For example, a callback function can be used to update the UI after data is retrieved from a server, or to execute a function after a user clicks a button.

Callbacks are commonly used in JavaScript for many purposes, including handling events, making asynchronous requests, and implementing iterators. It is a powerful tool for creating dynamic and responsive web applications.
